# 第一讲、计算机网络体系架构及交换技术

一、计算机网络体系架构

- 1、相互通信的两个计算机系统必须高度协调工作才行，而这种“协调”是相当复杂的。 “分层”可将庞大而复杂的问题，转化为若干较小的局部问题，而这些较小的局部问题就比较 易于研究和处理。 网络的体系结构 (Network Architecture) 是计算机网络的各层及其协议的集合，就是这个计算机 网络及其构件所应完成的功能的精确定义（不涉及实现）。
- 2、为了使不同体系结构的计算机网络都能互连，国际标准化组织 ISO 于 1977 年成立了专门 机构研究该问题。 1983年提出了一个试图使各种计算机在世界范围内互连成网的标准框架，即著名的开放系统互 连基本参考模型 OSI/RM (Open Systems Interconnection Reference Model)，简称为 OSI。 OSI参考模型定义了开放系统的层次结构、层次之间的相互关系，以及各层所包括的可能的服务。 OSI参考模型中的“开放”是指只要遵循OSI标准，一个系统就可以与位于世界上任何地方、同 样遵循同一标准的其它任何系统进行通信。
- 3、OSI参考模型将计算机网络从下到上划分为7个层次（因此也被称为OSI七层模型）


![image 1](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile1.png)

- （1）物理层（Physical Layer）是OSI的最低层，建立在物理通信介质的基础上，是系统和通信 介质的接口；物理层利用传输介质为通信的网络结点之间建立、管理和释放物理连接；实现二 进制比特流的透明传输，为数据链路层提供数据传输服务；物理层的数据传输单元是比特。
- （2）数据链路层（Data Link Layer）在物理层提供的比特流的传输服务基础上，在通信的相邻 结点的数据链路层对等实体间建立数据链路连接，并以“帧”为单位传输数据。 数据链路层的主要作用是采用差错控制、确认/重发以及流量控制等手段，将不可靠的有差错的 物理链路改造成无差错的数据链路。
- （3）网络层（Network Layer）通过路由算法将来自运输层的数据（报文）以分组为单位，在通 信子网中选择最适当的传输路径，将分组从发送端经过若干个中间节点传送到接收端 具体功能包括路由、拥塞控制、网际互连等 路由选择：网络层要解决分组如何跨越通信子网从发送端经过若干个中间结点传送到接收端 拥塞控制：为了避免通信子网中出现过多的分组而造成网络阻塞，需要对流入通信子网的分组 数量进行控制 网际互连：分组可能需要跨越多个通信子网才能到达目的地，网络层要解决这些通信子网如何 互连的问题
- （4）运输层（Transport Layer）称为端到端协议层，是负责数据通信的最高层，是位于面向网 络通信的低3层和面向信息处理的高3层的中间层。运输层在网络分层结构中起着承上启下的作 用，它利用低3层提供的服务向高3层提供可靠的端到端（end-to-end）的报文传输服务。 屏蔽 了各类通信子网的差异，向用户提供了一个通用的接口使得用户只需了解该接口，便可方便地 在任何网络上使用网络资源和进行通信。

- （5）会话层（Session Layer）是进程-进程的层次，应用层进程之间为完成某项处理任务需要进 行一系列内容相关的信息交换,会话层就是为有序、方便地控制这种进程间的信息交换提供控制 机制。会话层的主要功能：组织和同步不同主机上各种进程之间的通信（会话）。
- （6）表示层（Presentation Layer）为上层用户提供共同信息表示方式（语法）及信息表示方式 的变换。表示层的目的是处理有关被传输数据的表示问题；由于不同厂商的产品常使用不同的 信息表示标准，需要解决信息表示上的差异。 解决差异的方法：制定标准的数据表示格式（公共表示），并在保持数据含义的前提下进行数 据表示格式的转换，数据压缩 恢复、加密 解密也是表示层提供的表示变换功能。
- （7）应用层（Application Layer）是OSI参考模型中的最高层，为特定类型的网络应用提供访问 OSI环境的手段。应用层的目的是作为用户使用OSI功能的唯一窗口，应用层是功能最丰富、实 现最复杂的一层，同时又是相对最不成熟的一层该层包含了许多服务，有的服务标准已经出台， 更多的服务标准还没有出台。


- 4、OSI 参考模型把对等层次之间传送的数据单位称为该层的协议数据单元 PDU (Protocol Data Unit)。这个名词现已被许多非 OSI 标准采用。 任何两个同样的层次把数据（即数据单元加上控制信息）通过水平虚线直接传递给对方。这就 是所谓的“对等层”(peer layers)之间的通信。 各层协议实际上就是在各个对等层之间传递数据时的各项规定。
- 5、OSI参考模型本身并不是网络体系结构 按照定义，网络体系结构是网络层次结构模型和各层协议的集合。 OSI参考模型仅仅给出了各层的协议规范，只是描述了每一层的功能，并没有实现各层的协议。
- 6、TCP/IP 体系结构 TCP/IP 的全称是传输控制协议/互联网络协议（Transmission Control Protocol/Internet Protocol）。 TCP/IP 协议是协议栈，包括了很多的协议，并没有提出参考模型； 目前我们使用的是版本 4，它的网际层 IP 协议一般记作 IPv4 ； 版本 6 的网际层 IP 协议一般记作 IPv6（或 IPng, IP next generation）。
- 7、TCP/IP 参考模型特点

 开放的协议标准;

 独立于特定的计算机硬件与操作系统；

 独立于特定的网络硬件，可以运行在局域网、广域网，更适用于互连网中；

 统一的网络地址分配方案，使得整个 TCP/IP 设备在网中都具有唯一的地址；

 标准化的高层协议，可以提供多种可靠的用户服务。

- 8、TCP/IP 分层


![image 3](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile3.png)

- （1）网络接口层（Host-to-Network Layer）


TCP/IP 参考模型的最低层，它负责通过网络发送和接收数据。 网络接口层并没有规定具体的协议，它采取开放的策略，允许使用广域网、局域网与城域网的 各种协议，如 Ethernet、令牌、帧中继、ATM 等。 任何一种流行的低层传输协议都可以与 TCP/IP 协议网际层接口。这正体现了 TCP/IP 协议体系 的开放性、兼容性的特点，也是 TCP/IP 协议成功应用的基础。 网络接口层的协议数据单元为帧、比特。

- （2）网际层（Internet Layer） TCP/IP 参考模型网际层的协议是 IP 协议。 IP 协议是一种不可靠、无连接的数据报传送服务协议，它提供的是一种“尽力而为”（best-effort） 的服务，相当于 OSI 参考模型网络层无连接的网络服务。 网际层的协议数据单元是 IP 分组。 主要功能有：分组封装与解封、IP 分组的路由、流量控制与拥塞控制
- （3）运输层（Transport Layer） 负责在会话进程之间建立和维护用于报文传输的端到端的传输连接。 定义了两种不同的协议：传输控制协议 TCP 与用户数据报协议 UDP。 TCP 是一种可靠的、面向连接、面向字节流的运输层协议。TCP 协议提供比较完善的流量控制 与拥塞控制功能。 UDP 是一种不可靠的、无连接的运输层协议。
- （4）应用层（Application Layer） 应用层是 TCP/IP 参考模型的最高层 该层包含各种标准的应用层协议，为特定类型的网络应用提供相应的服务。


![image 4](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile4.png)

- 9、OSI 与 TCP/IP 的简单比较 OSI 参考模型产生在 OSI 参考模型的协议发明之前，这就意味着该模型没有偏向与任何特定的 协议，因此具有非常好的通用性。 TCP/IP 模型却正好相反，首先出现的是 TCP/IP 协议， TCP/IP 模型实际上是参照 OSI 参考模型 对已有的 TCP/IP 协议的描述。因此 TCP/IP 模型只适用于 TCP/IP 协议，不适合于任何其它的非 TCP/IP 协议。

二、交换技术

- 1、在通信系统中，为了解决大量计算机相互通信的问题，在任意两个要进行通信的设备之间建 立一条临时的(实际的或虚拟的)通信链路来传输通信数据的技术就是数据交换（Data Switching） 技术。从通信资源的分配角度来看，就是按照某种方式动态地分配传输线路的资源。
- 2、电路交换（Circuit Switching）也称为电话交换（线路交换），它为一对需要进行通信的节点 之间建立一条临时的物理的专用的通信线路，通信完毕后，通信线路即被拆除。电路交换技术 主要应用于通过电话线路的通信。


- （1）电路交换过程 分为三个阶段：


 建立连接：建立一条专用的物理通路（占用通信资源）。

 通话：主叫和被叫双方互相通电话（一直占用通信资源）。

 释放连接：释放刚才使用的专用的物理通路（归还通信资源）。

- （2）电路交换特点：通话的两个用户始终占用端到端的通信资源。 计算机数据通信具有突发性，这导致在传送数据时，通信线路的利用率很低，真正用来传送数 据的时间往往不到 10%，甚至不到 1%，已被用户占用的通信线路资源在绝大部分时间里都是 空闲的。
- （3）电路交换的优点：数据传输可靠，迅速，而且保证顺序，适合远距离成批传输数据和实时 性要求高的场合。
- （4）电路交换的缺点：


- （1）不适合计算机之间的数据传输
- （2）不具备数据存储能力，不能平滑通信量
- （3）误码率较高，无法发现与纠正传输差错
- （4）线路的利用率低、通信成本高


- 3、存储转发技术 发送的数据与目的地址、源地址、控制信息按照一定格式组成一个数据单元(报文或报文分组) 进入通信子网；通信子网中的节点是路由器，它负责完成数据单元的接收、差错校验、存储、 路径选择和转发功能。


![image 7](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile7.png)

- （1）存储转发技术优点

 由于通信子网中的路由器可以存储数据单元，多个数据单元可以共享通信信道，线路利 用率高；

 通信子网中路由器有路由选择功能，可以动态选择数据单元通过通信子网的最佳路径；

 数据单元在通过通信子网中的每个路由器时，均要进行差错检查与纠错处理，因此可以 减少传输错误，提高系统可靠性；

 通过路由器可以对不同通信速率的线路进行转换，也可以对不同的数据代码格式进行变 换。

- （2）存储转发技术方式分为

 报文交换(message switching)

 报文分组交换，简称分组交换(packet switching)

- （3）报文交换

 报文交换起源于电报；

 报文交换的数据基本单位是长度不固定的报文；

 每一个报文由传输的数据和报头组成，报头中有源地址和目的地址。路由器根据报头中 的目的地址以及自己的转发表为报文进行路径选择，并且对收发的报文进行相应的处理。

- （4）报文交换的优点：

 与电路交换相比，报文交换没有建立电路和拆除通信线路所需的等待和时延；

 通信线路利用率高，节点间可根据通信线路情况选择不同的速度传输，能高效地传输数

- （5）报文交换的缺点：

 长报文传输时间比较长。

 长报文出错的可能性较大，出错重传所花费时间较长。

 由于采用了对大报文的存储/转发，节点存储 转发的时延较大，不适用于交互式通信。

- （6）分组交换 报文交换的缺点是由报文太长和长短不固定引起的，因此分组交换的思想是限制发送和转发的 固定数据长度。 分组交换将一个大报文分割成多个一定长度的数据段，并为每个数据段加上分组头（其中包括 源、目标地址信息以及分组信息），形成一个分组（Packet），然后以分组为单位存储转发，最 后在接收端再将各分组重新组装成一个完整的报文。

- （7）分组在互联网中的转发

 分组交换在传送数据之前不必先占用一条端到端的通信资源。分组在哪段链路上传送才 占用这条链路的通信资源。

 根据首部中包含的目的地址、源地址等重要控制信息进行转发。

 每一个分组在互联网中独立选择传输路径。

 位于网络核心部分的路由器负责转发分组，即进行分组交换。

 路由器要创建和动态维护转发表。

- （8）分组交换的优点

 高效：动态分配传输带宽，对通信链路是逐段占用。

 灵活：以分组为传送单位和查找路由。

 迅速：不必先建立连接就能向其他主机发送分组。

 可靠：保证可靠性的网络协议；分布式的路由选择协议使网络有很好的生存性。

- （9）分组交换的缺点

 分组在各节点存储转发时需要排队，这就会造成一定的时延。

 分组必须携带的首部（里面有必不可少的控制信息）也造成了一定的开销。

 无法确保通信时端到端所需的带宽，在通信量较大时可能造成网络拥塞。

 分组到达目的主机时会出现丢失、重复与乱序的现象。

 使用运输层端口号的 NAT 叫做网络地址与端口号转换 NAPT (Network Address and Port Translation)，而不使用端口号的 NAT 就叫做传统的 NAT (traditional NAT)。

- 3、RIP协议


- （1）RIP 的工作原理

 路由信息协议 RIP (Routing Information Protocol) 是一种分布式的、基于距离向量的路 由选择协议。

 要求网络中的每个路由器都要维护从它自己到其他每一个目的网络的距离记录。 RIP“距离”的定义

 路由器到直接连接的网络的距离 = 1。

 路由器到非直接连接的网络的距离 = 所经过的路由器数 1。

 RIP 协议中的“距离”也称为“跳数”(hop count)，每经过一个路由器，跳数就加 1。

 好路由 = “距离短”的路由。最佳路由 = “距离最短”的路由。

 一条路径最多只能包含 15 个路由器。

 “距离”的最大值为 16 时即相当于不可达。

 RIP 不能在两个网络之间同时使用多条路由，只选择距离“最短”的路由。

- （2）RIP 协议的三个特点


- 1. 仅和相邻路由器交换信息。
- 2. 交换的信息是当前本路由器所知道的全部信息，即自己的路由表。
- 3. 按固定时间间隔交换路由信息，例如，每隔 30 秒。为加快协议的收敛速度，当网络拓 扑发生变化时，路由器也及时向相邻路由器通告拓扑变化后的路由信息（即触发更新）。


- （3）路由表的建立

 路由器在刚刚开始工作时，路由表是空的。

 然后，得到直接连接的网络的距离（此距离定义为 1）。

 之后，每一个路由器也只和数目非常有限的相邻路由器交换并更新路由信息。

 经过若干次更新后，所有的路由器最终都会知道到达本自治系统中任何一个网络的最短 距离和下一跳路由器的地址。

 RIP 协议的收敛 (convergence) 过程较快。“收敛”就是在自治系统中所有的结点都得 到正确的路由选择信息的过程。

- 4、OSPF协议


- （1）内部网关协议 OSPF

 开放最短路径优先 OSPF (Open Shortest Path First)是为克服 RIP 的缺点在 1989 年开 发出来的。

 “开放”表明 OSPF 协议不是受某一家厂商控制，而是公开发表的。

 使用了 Dijkstra 提出的最短路径算法 SPF。

 采用分布式的链路状态协议 (link state protocol)。

- （2）三个主要特点


 采用洪泛法 (flooding)，向本自治系统中所有路由器发送信息。

 发送的信息是与本路由器相邻的所有路由器的链路状态，但这只是路由器所知道的部分 信息。

 链路状态：说明本路由器都和哪些路由器相邻，以及该链路的度量 (metric)。

 当链路状态发生变化或每隔一段时间（如 30 分钟），路由器才用洪泛法向所有路由器发 送此信息。

- （3）链路状态数据库

 由于各路由器之间频繁地交换链路状态信息，因此所有的路由器最终都能建立一个链路 状态数据库。

 每个路由器使用链路状态数据库中的数据构造自己的路由表。

- （4）OSPF 的区域(area)

 为了使 OSPF 能够用于规模很大的网络，OSPF 将一个自治系统再划分为若干个更小 的范围，叫作区域。

 每一个区域都有一个 32 位的区域标识符（用点分十进制表示）。

 区域也不能太大，在一个区域内的路由器最好不超过 200 个。

 OSPF 使用层次结构的区域划分。在上层的区域叫作主干区域(backbone area)。主干区 域的标识符规定为 0.0.0.0。主干区域的作用是用来连通其他在下层的区域。

![image 56](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile56.png)

 OSPF 对不同的链路可根据 IP 分组的不同服务类型 TOS 而设置成不同的代价。因此， OSPF 对于不同类型的业务可计算出不同的路由。

 如果到同一个目的网络有多条相同代价的路径，那么可以将通信量分配给这几条路径。 这叫作多路径间的负载平衡。

 所有在 OSPF 路由器之间交换的分组都具有鉴别的功能。

 支持可变长度的子网划分和无分类编址 CIDR。

 32 位的序号，序号越大状态就越新。全部序号空间在 600 年内不会产生重复号。

- 1. 问候 (Hello) 分组。
- 2. 数据库描述 (Database Description) 分组。
- 3. 链路状态请求 (Link State Request) 分组。
- 4. 链路状态更新 (Link State Update) 分组。
- 5. 链路状态确认 (Link State Acknowledgment)分组。

- 1、UDP


- （1）UDP 在 IP 的数据报服务之上增加了的功能：复用和分用和差错检测的功能
- （2）UDP 的主要特点

- 1) UDP 是无连接的，发送数据之前不需要建立连接，，因此减少了开销和发送数据之前的 时延。
- 2) UDP 使用尽最大努力交付，即不保证可靠交付，因此主机不需要维持复杂的连接状态 表。
- 3) UDP 是面向报文的。UDP 对应用层交下来的报文，既不合并，也不拆分，而是保留这 些报文的边界。UDP 一次交付一个完整的报文。

 发送方 UDP 对应用程序交下来的报文，在添加首部后就向下交付 IP 层。UDP 对应

用层交下来的报文，既不合并，也不拆分，而是保留这些报文的边界。  应用层交给 UDP 多长的报文，UDP 就照样发送，即一次发送一个报文。

 接收方 UDP 对 IP 层交上来的 UDP 用户数据报，在去除首部后就原封不动地交付上 层的应用进程，一次交付一个完整的报文。

 应用程序必须选择合适大小的报文。

若报文太长，UDP 把它交给 IP 层后，IP 层在传送时可能要进行分片，这会降低 IP 层 的效率。

- 5) UDP 支持一对一、一对多、多对一和多对多的交互通信。
- 6) UDP 的首部开销小，只有 8 个字节，比 TCP 的 20 个字节的首部要短。

- 1. 当 cwnd < ssthresh 时，使用慢开始算法。
- 2. 当 cwnd > ssthresh 时，停止使用慢开始算法，改用拥塞避免算法。

三、中继器

中继器是位于 OSI 参考模型的物理层的网络设备，常用于两个网络节点之间物理信号的双 向转发工作，是网络互连中最简单的设备。采用中继器所连接的网络，在逻辑功能方面实际上 是同一个网络。

![image 132](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile132.png)

![image 133](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile133.png)

由于存在损耗，在线路上传输的信号功率会逐渐衰减，衰减到一定程度时将造成信号失真， 因此会导致接收错误。中继器就是为解决这一问题而设计的。它完成物理线路的连接，对衰减 的信号进行再生，保持与原数据相同。

一般情况下，中继器的两端连接的是相同的媒体，但有的中继器也可以完成不同媒体的转 接工作。从理论上讲中继器的使用是无限的，网络也因此可以无限延长。事实上这是不可能的， 因为网络标准中都对信号的延迟范围作了具体的规定，中继器只能在此规定范围内进行有效的 工作，否则会引起网络故障。

中继器不关心数据的格式和含义，它只负责复制和增强通过物理介质传输的表示“1”和“0” 的信号。

中继器的主要功能是扩展一个工作站或一组工作站与网络中其它部分的距离。

- （1）中继器的特点：


 中继器可以将局域网的一个网段和另一个网段相连，而且可以连接不同类型的介质。

六、交换机

1990 年问世的交换式集线器(switching hub)，可明显地提高局域网的性能。交换式集线器 常称为以太网交换机(switch)或第二层交换机（表明此交换机工作在数据链路层）。实质上是一 个多接口网桥。通常有十几个或更多的接口。每个接口都直接与一个单台主机或另一个以太网 交换机相连，并且一般都工作在全双工方式。

- 1、特点：

 以太网交换机具有并行性。

 能同时连通多对接口，使多对主机能同时通信。

 相互通信的主机都独占传输媒体，无碰撞地传输数据。

 每一个端口和连接到端口的主机构成了一个碰撞域。

- 2、以太网交换机的接口有存储器，能在输出端口繁忙时把到来的帧进行缓存。交换机是一种即 插即用设备。其内部的帧交换表（又称为地址表）是通过自学习算法自动地逐渐建立起来的。 这种交换表就是一个内容可寻址存储器 CAM (Content addressable Memory)。 使用专用的交换结构芯片，用硬件转发，其转发速率要比使用软件转发的网桥快很多。

 下一跳地址（Gw）

 发送的物理端口（interface）

 路由信息的来源（Owner）

 路由优先级（pri）

 度量值（metric）

# 第五讲、网络安全

一、网络安全问题概述

- 1、计算机网络面临的安全性威胁 两大类威胁：被动攻击和主动攻击。

![image 207](0815-国网通信-数据通信网-蔡老师-暑期基础_images/imageFile207.png)

- （1）、被动攻击

 指攻击者从网络上窃听他人的通信内容。

 通常把这类攻击称为截获。

 攻击者只是观察和分析某一个协议数据单元 PDU，以便了解所交换的数据的某种性质，但 不干扰信息流。

 这种被动攻击又称为流量分析 (traffic analysis)。

- （2）、主动攻击


 篡改：故意篡改网络上传送的报文。这种攻击方式有时也称为更改报文流。

 恶意程序 (rogue program)：种类繁多，主要包括：计算机病毒、计算机蠕虫、特洛伊木马、 逻辑炸弹、后门入侵、流氓软件等。

 拒绝服务 DoS (Denial of Service)：指攻击者向互联网上的某个服务器不停地发送大量分组， 使该服务器无法提供正常服务，甚至完全瘫痪。

 分布式拒绝服务 DDoS：若从互联网上的成百上千的网站集中攻击一个网站，则称为分布 式拒绝服务 DDoS (Distributed Denial of Service)。有时也把这种攻击称为网络带宽攻击或 连通性攻击。

- 2、计算机网络通信安全的目标

 对于主动攻击，可以采取适当措施加以检测。

 对于被动攻击，通常是检测不出来的。 （1）计算机网络通信安全的目标：

- 1）防止分析出报文内容和流量分析。
- 2）防止恶意程序。
- 3）检测更改报文流和拒绝服务。


 对付被动攻击：加密技术。

- 1、网络层安全协议 IP 几乎不具备任何安全性，不能保证：1.数据机密性 2.数据完整性 3.数据来源认证。

 封装安全有效载荷 ESP (Encapsulation Security Payload) 协议，提供源点鉴别、数据完 整性和保密。

- 3）互联网密钥交换 IKE (Internet Key Exchange) 协议

 适合于主机到主机之间的安全传送。

 需要使用 IPsec 的主机都运行 IPsec 协议

- 2）隧道方式 (tunnel mode)


 在原始的 IP 数据报的前后分别添加若干控制信息，再加上新的 IP 首部。

 需要在 IPsec 数据报所经过的所有路由器上都运行 IPsec 协议。

 隧道方式常用来实现虚拟专用网 VPN。

- （4）安全关联：在发送 IP 安全数据报之前，在源实体和目的实体之间必须创建一条网络层的 逻辑连接。此逻辑连接叫做安全关联 SA (Security Association) 。 安全关联的特点：


 安全关联是从源点到终点的单向连接，它能够提供安全服务。

 在安全关联 SA 上传送的就是 IP 安全数据报。

 如要进行双向安全通信，则两个方向都需要建立安全关联。

 若 n 个员工进行双向安全通信，一共需要创建 (2 + 2n ) 条安全关联 SA。

 对打上标签的 IP 数据报在第二层（链路层）用硬件进行转发。

 采用硬件技术对打上标签的 IP 数据报进行转发就称为标签交换。

 可以使用多种数据链路层协议，如 PPP、以太网、ATM 以及帧中继等。

- （2）MPLS 域 (MPLS domain) ：指该域中有许多彼此相邻的路由器，并且所有的路由器都是 支持 MPLS 技术的标记交换路由器 LSR (Label Switching Router)。 LSR 同时具有标记交换和路由选择这两种功能。标记交换功能是为了快速转发，路由选择功能 是为了构造转发表。


- 2、MPLS 基本工作过程

- （1）找出标签交换路径 LSP。

 各 LSR 使用标签分配协议 LDP (Label Distribution Protocol) 交换报文，找出和标签相对应 的标签交换路径 LSP (Label Switched Path)。整个标签交换路径就像一条虚连接一样。

- （2）打标签，然后转发。

 入口节点 (ingress node) 给进入 MPLS 域的 IP 数据报打上标签（实际上是插入一个 MPLS 首部），并按照转发表把它转发给下一个 LSR。以后的所有 LSR 都按照标签进行 转发。

 给 IP 数据报打标签的过程叫做分类 (classification)。

- （3）标签对换。

 一个标签仅在两个 LSR 之间才有意义。

 LSR 要做两件事：转发，更新标记。更新标记：把入标记更换成为出标记。称之为标签对 换 (label swapping)。

- （4）去除标签。


 当分组离开 MPLS 域时，MPLS 出口节点 (egress node) 把分组的标签去除。

 把 IP 数据报交付给非 MPLS 的主机或路由器。
